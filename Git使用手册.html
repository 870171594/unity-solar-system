🚀 Unity项目 Git 使用手册

简单易懂的版本控制指南 | 团队协作必备

⚡ 快速开始

记住这4个命令就够了！
📥 更新代码
git pull
➕ 添加修改
git add .
💾 提交说明
git commit -m "..."
📤 上传代码
git push

🎯 第一次使用（只需做一次）

步骤1：安装软件

1 下载安装 Git for Windows
2 下载安装 Unity 2021.3

步骤2：克隆项目

# 打开命令行（按Win+R，输入cmd，回车） git clone https://github.com/870171594/unity-solar-system.git
项目会下载到当前目录，完成后用Unity打开 unity-solar-system 文件夹即可

步骤3：配置个人信息（只需一次）

# 设置你的名字和邮箱 git config user.name "你的名字" git config user.email "你的邮箱"

🔄 日常使用流程

每次工作前先 git pull，工作完成后 git push

📥 开始工作前

1 打开命令行，进入项目文件夹
cd unity-solar-system
2 更新到最新代码
git pull
这一步很重要！可以避免后面的冲突问题

📤 完成工作后

1 添加所有修改的文件
git add .
2 提交修改（写清楚你做了什么）
git commit -m "修复了相机控制bug"
3 上传到GitHub
git push
完成！其他成员现在可以看到你的修改了

📚 常用命令速查

操作 命令 说明
查看状态 git status 查看哪些文件被修改了
查看历史 git log 查看提交记录
放弃修改 git checkout -- 文件名 撤销对某个文件的修改
查看差异 git diff 查看具体改了什么

❓ 常见问题解答

🔴 出现 "Please tell me who you are" 怎么办？

这是你第一次使用Git，需要设置身份信息：

git config user.name "你的名字" git config user.email "你的邮箱"
🔴 推送时要求输入账号密码？

输入你的GitHub账号和密码即可。

如果经常推送，建议设置SSH密钥，这样就不用每次都输入密码了

🔴 出现冲突怎么办？

当两个人同时修改同一个文件时会出现冲突：

git pull

Git会标记冲突的地方，打开文件会看到：

<<<<<<< HEAD
你的修改
=======
别人的修改
>>>>>>> origin/main

不要慌！找负责人商量保留哪部分，然后删除标记符号

🔴 提交后发现写错了怎么办？

如果还没有push，可以修改最后一次提交：

git commit --amend -m "新的提交信息"

如果已经push了，就不要用这个命令！直接再提交一个修改

🔴 Unity打开后Library文件夹很大，要不要提交？

不需要！Library文件夹已经在 .gitignore 中被忽略了

Unity会自动重新生成这个文件夹，每个人打开项目时都会自动创建

⚠️ 重要注意事项

不要随意强制推送！ git push -f 会覆盖别人的代码
提交前先更新！ 每次开始工作前先执行 git pull
写清楚的提交信息！ 不要写 "update"、"fix" 这种模糊的词
好的提交信息示例：
  • ✅ "添加了FPS设置UI界面"
  • ✅ "修复相机旋转时的卡顿问题"
  • ✅ "更新月球材质贴图"
场景文件容易冲突！ .unity文件是二进制格式，很难合并
建议：场景和Prefab尽量由一个人负责编辑

👥 团队协作建议

推荐的工作流程：

1 开始工作前：先在群里说一声你要修改什么
2 执行 git pull 更新代码
3 专心完成你的工作
4 完成后：先 git pull 再 git push
5 在群里告诉大家你完成了什么

遇到问题时：

不要自己瞎搞！ 先问负责人或资深成员
不确定的命令不要执行！ 尤其是带 --force 的命令

💜 有问题随时联系项目负责人
